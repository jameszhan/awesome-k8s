# 知识地图（Knowledge Map）

## 元属性

- 定义（Definition）: 提供了一个清晰、准确的描述，用于确立元素或概念的基本意义。
- 内涵（Intension）: 指的是元素或概念本身固有的属性或特性，这些属性或特性使其成为一个独特的类别。
- 外延（Extension）: 指的是元素或概念可以应用或延伸到的具体实例或情境。
- 用途（Utility or Use）: 描述了该元素或概念在实际应用中的功能或价值。

## 基础性元素(节点)

| 名称                         | 定义 | 用途                                                | 内涵 | 外延          |
|----------------------------|-|---------------------------------------------------|-|-------------|
| 概念（Concepts）| 基础的、抽象的思想或类别 | 通常作为其他元素的基础<br />定义和界定一个特定领域或问题的基础思想或类别<br />提供主题或领域的基础理解 | 表达一个抽象思想或者分类的基础概念 | 如“自由”、“正义”等 |
| 事实（Facts）| 具体、可验证的信息或陈述 | 通常用于支持或解释概念<br />具体、可验证的信息或陈述<br />提供具体和可验证的信息 | 具有明确真值的信息或声明 | 如“地球绕太阳转”   |
| 概念和主题（Concepts and Themes） | | 这是最基础的元素，通常用作其他元素的出发点<br />作为知识地图的核心，提供主题或领域的基础理解 | |
| 事实和数据（Facts and Data）      | | 这些提供了对概念和问题的具体和可验证的信息<br />提供具体和可验证的信息，通常用于支持或解释概念和问题 | |

## 动态性元素(节点)

| 名称       | 定义 | 用途                                | 内涵 | 外延 |
|----------|-|-----------------------------------|-|-|
| 问题（Questions）| 需要解决或回答的疑问或难题 | 通常是知识地图的一个重要驱动力<br />这通常是知识地图的出发点，即你希望通过知识地图来解答或解决什么问题<br />作为知识地图的一个重要驱动力，提出需要解决或回答的疑问或难题<br />识别需要解决的关键问题 | 表示对信息、状况或现象的疑问或不明确性 | |
| 解决方案（Solutions）| 针对特定问题的答案或方法 | 通常需要综合应用多种知识元素/针对特定问题提供的答案或方法<br />这是知识地图的一部分目标，即对于识别出的问题，知识地图应该能够提供或引导到一个或多个解决方案<br />通常需要综合应用多种知识元素<br />提供针对已识别问题的答案或方法 | 针对一个问题的解决方法或方案 | 需要综合应用多种知识元素 |
| 问题和解决方案（Questions and Solutions） | | 特别是在教育或企业环境中，解决特定问题通常是知识地图创建的一个重要驱动因素<br />识别需要解决的关键问题和可能的解决方案 | |
| 流程（Processes） | 为了达到某个目标或解决某个问题而需要遵循的活动序列 | 一系列有逻辑关系的步骤或活动<br />描述为达成某一目标而需要遵循的活动序列<br />描述完成任务或达成目标所需的步骤和流程 | 描述一系列有逻辑关系的活动或任务 | 如生产流程、决策流程等 |
| 步骤（Steps）| | 流程中的单个活动或任务<br />具体化流程，提供可操作的活动清单 | |
| 流程和步骤（Processes and Steps） | | 为了达到某个目标或解决某个问题而需要遵循的活动序列<br />某些知识地图会明确表示完成某项任务或达成某个目标所需的步骤和流程<br />描述为达成某一目标或解决某一问题而需要遵循的活动序列<br />尤其是在工程或业务流程管理中<br />在工程或业务流程管理中，描述完成任务或达成目标所需的步骤和流程 | |
